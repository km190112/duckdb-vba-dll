dackdb 使い方マニュアル

Excel の VBA から DuckDB のデータベース（dack.db）を読み書きするためのモジュールです。数十万行を 1 秒かからずシートに展開できます。

対象 64bit 版 Excel 用意するもの DLL 1 ファイルと .bas 1 ファイル 前提知識 VBA の基本

はじめにこれは何か

DuckDB は 1 つのファイルにデータを収める分析用データベースです。Excel の 104 万行の壁を超える量のデータを置いておき、必要な分だけ SQL で絞り込んでシートに取り出す、という使い方ができます。

このモジュールは、その DuckDB を VBA から直接呼び出せるようにするものです。配布された DLL によって、あなたができる操作の範囲が決まります。

💡

日本語がそのまま扱えます。テーブル名・列名・データのいずれにも日本語を使えて、文字化けしません。𠮷 のようなサロゲートペアや絵文字も含みます。ODBC 接続を使った方法だとここが壊れますが、このモジュールでは起きません。

ステップ 1準備する

初回だけ必要な作業です。3 つのステップで終わります。

  1. ファイルを置く

    配布された DLL を、ブックと同じ場所の dll フォルダに入れます。フォルダ名や場所は自由ですが、以下の説明はこの配置を前提にします。

    売上管理.xlsm
    dll\dackdb_r.dll　　← 配布された DLL を 1 つ

  2. モジュールを読み込む

    Excel で Alt + F11 を押して VBE を開き、メニューの「ファイル」→「ファイルのインポート」から、DLL に対応する .bas ファイルを選びます。

    配布された DLL読み込む .bas
    dackdb_r.dllDackDbR.bas
    dackdb_rw.dllDackDbRW.bas
    dackdb_admin.dllDackDbAdmin.bas

  3. ブックを開いたときに初期化する

    VBE の左側で「ThisWorkbook」をダブルクリックし、次のコードを貼り付けます。ブックを開くたびに自動で実行されます。

    Private Sub Workbook_Open()
    DackDbInit ThisWorkbook.Path & "\dll"
End Sub

    これで準備完了です。ブックを保存して開き直してください。

⚠️

32bit 版の Excel では動きません。「ファイル」→「アカウント」→「Excel のバージョン情報」で確認できます。32bit 版だった場合は初期化時にその旨のエラーが出ます。

ステップ 2自分の権限を確認する

DLL は 3 種類あり、できる操作が違います。下のボタンで、あなたが配布された DLL を選んでください。このページ全体が、その権限でできることに合わせて表示されます。

分からない場合は、次のコードを実行すると自分の権限が表示されます。

Sub 権限を確認()
    MsgBox DackVersion()
End Sub

' 表示例:
' dackdb 0.1.0 / DuckDB v1.5.5 / 権限: READ_WRITE (読み書き可（DDL 不可）)
📝

権限外の操作を書いても、VBA のコードとしては正しく書けます。実行したときに「どの DLL を使えばよいか」を伝えるエラーが出ます。上位の DLL を渡されれば、コードを 1 文字も直さずそのまま動きます。

しくみDLL を呼ぶしくみ

ふだんは意識しなくて構いませんが、エラーの原因を追うときや自分で関数を足したいときのために、Excel が DLL を呼ぶ流れを説明します。

VBA から DLL の関数を呼ぶ宣言

VBA は Declare 文で DLL の中の関数を「VBA の関数として」使えるようにします。モジュールの先頭にこの宣言が並んでいます。

Private Declare PtrSafe Function DackQuery Lib "dackdb_r.dll" ( _
    ByVal handle As LongLong, _
    ByVal sqlPtr As LongPtr, _
    ByRef result As Variant) As Long
  • PtrSafe — 64bit 環境で使う宣言であることを示します。これが無いと 64bit Excel でエラーになります
  • Lib "dackdb_r.dll" — どの DLL の関数かを指定します。この文字列だけが 3 つのモジュールの違いです
  • 戻り値の Long — 0 なら成功、負の数ならエラーです

なぜ DLL のフォルダを DackDbInit で教えるのか

Lib "dackdb_r.dll" はファイル名だけで、場所を書いていません。Windows は決められた場所しか探さないので、そのままではブックの隣にある DLL を見つけられません。

そこで DackDbInit が、先回りしてフルパスで DLL を読み込んでおきます。一度読み込まれた DLL は名前で参照できるようになるため、以降の Declare がすべて解決されます。

' DackDbInit の中身（抜粋）
Private Declare PtrSafe Function LoadLibraryW Lib "kernel32" ( _
    ByVal lpFileName As LongPtr) As LongPtr

LoadLibraryW StrPtr("C:\...\dll\dackdb_r.dll")   ' フルパスで先に読み込む

これがブックを開くたびに一度だけ必要な理由です。フォルダを移動したりブックを別の場所にコピーした場合も、パスさえ合っていればそのまま動きます。

文字列の受け渡し：日本語が化けない理由

💡

VBA の Declare は String 型を必ず ANSI（日本語環境では CP932）に変換してしまいます。これがふつうにやると日本語が化ける原因です。

そこでこのモジュールは String を直接渡さず、StrPtr() で文字列そのものの置き場所（メモリ上の位置）を数値として渡しています。VBA は数値を変換しないので、Unicode のまま DLL に届きます。

' ✗ これだと VBA が勝手に CP932 に変換して化ける
'   Declare ... (ByVal sql As String)

' ✓ 位置だけを渡すので変換されない
'   Declare ... (ByVal sqlPtr As LongPtr)
DackQuery handle, StrPtr(sql), result

戻り側も同じ理屈で、String ではなく Variant で受け取ります。Variant は VBA が変換せずそのまま扱うため、日本語も配列も安全に渡せます。

成功と失敗が同じ形で返る

すべての関数が「戻り値で成否、Variant で中身」という同じ形をしています。

戻り値意味result の中身
0成功結果（配列・数値・文字列）
-1一般的なエラー日本語のエラーメッセージ
-403権限エラー必要な DLL を案内するメッセージ
-999内部エラー（不具合）詳細。報告してください

モジュール内の Check という処理がこれを見て、エラーなら VBA のエラー（Err.Raise）に変換します。だから使う側は On Error でふつうに捕まえられます。

基本データを取り出す

いちばん短い例

データベースを開き、SQL の結果をシートに貼り、閉じる。これが基本形です。

Sub 営業部の売上を取得()
    Dim h As LongLong
    h = OpenDb("C:\data\dack.db")

    Dim n As Long
    n = QueryToSheet(h, "SELECT * FROM 売上 WHERE 部署 = '営業部'", _
                     Sheet1.Range("A1"))

    CloseDb h
    MsgBox n & " 件貼り付けました"
End Sub

QueryToSheet は 1 行目に列名、2 行目以降にデータを貼り付け、データの行数（列名の行を除いた数）を返します。10 万行でも一瞬で終わります。

🔒

CloseDb を忘れないでください。閉じないとファイルがロックされたままになり、他の人がそのデータベースを開けなくなります。エラーが起きても必ず閉じたい場合は、下の「安全な書き方」を参考にしてください。

配列として受け取る

シートに貼らず、VBA の中で処理したいときは Query を使います。

Dim arr As Variant
arr = Query(h, "SELECT 部署, 金額 FROM 売上")

' arr(1, 1) = "部署"　　← 1 行目は必ず列名
' arr(1, 2) = "金額"
' arr(2, 1) = "営業部"　← 2 行目からがデータ
' arr(2, 2) = 1000

Dim i As Long, 合計 As Double
For i = 2 To UBound(arr, 1)
    If Not IsNull(arr(i, 2)) Then 合計 = 合計 + arr(i, 2)
Next

データが 0 件のときは列名の行だけが返るので、UBound(arr, 1) = 1 で判定できます。データベースの NULL は IsNull() で判定でき、シートに貼ると空欄になります。

シートの値を条件に使うとき

🛡️

SQL に文字列を連結しないでください。セルに ' OR 1=1; DROP TABLE 売上; -- のような値が入っていると、意図しない SQL が実行されてしまいます。

代わりに QueryP を使い、値を ? の位置に渡します。渡した値は必ず「ただの値」として扱われ、SQL として解釈されることはありません。

' ✗ 危険：セルの内容が SQL の一部になってしまう
arr = Query(h, "SELECT * FROM 売上 WHERE 部署 = '" & Range("B1").Value & "'")

' ✓ 安全：? に値として渡る
arr = QueryP(h, "SELECT * FROM 売上 WHERE 部署 = ? AND 金額 > ?", _
             Array(Range("B1").Value, 1000))

渡し方は Array(値1, 値2) でも Range("A1:C1").Value でも構いません。縦並びの範囲（Range("A1:A3").Value）も受け付けます。

安全な書き方（エラーが起きても必ず閉じる）

Sub 売上を取得()
    Dim h As LongLong
    On Error GoTo 後始末

    h = OpenDb("C:\data\dack.db")
    QueryToSheet h, "SELECT * FROM 売上", Sheet1.Range("A1")

後始末:
    If h <> 0 Then CloseDb h
    If Err.Number <> 0 Then MsgBox Err.Description, vbExclamation
End Sub

基本データを書き込む

🔒 読み取り専用の DLL では、この章の操作はできません。dackdb_rw.dll が必要です。

シートの範囲をまとめて追加する

Excel から DB へデータを入れる、いちばん速い方法です。2 万行で 0.04 秒ほどで終わります。

Dim n As LongLong
n = AppendRange(h, "売上", Sheet1.Range("A2:D10000").Value)

使うときの決まりは 3 つです。

  • 見出しの行は含めない。データの行だけの範囲を渡します（上の例では 1 行目が見出しなので A2 から）。
  • 列の並び順をテーブルに合わせる。1 列目がテーブルの 1 列目に入ります。列数が違うとエラーになります。
  • 途中で失敗したら 1 行も入りません。「1000 行目でエラー、でも 999 行は入った」という中途半端な状態にはなりません。
⚠️

範囲に #N/A や #DIV/0! などのエラー値のセルがあると、その位置を示してエラーになります（例:「3 行 2 列目: セルがエラー値です」）。空欄として扱いたい場合は IFERROR で空にしてから渡してください。エラー値を勝手に空欄に変えてしまうと、気づかないうちにデータが壊れるためです。

1 件ずつ追加・更新・削除する

' 影響した行数が返る
Dim n As LongLong

n = Exec(h, "DELETE FROM 売上 WHERE 日付 < DATE '2020-01-01'")

' セルの値を使うときは ExecP（? に値が入る）
n = ExecP(h, "UPDATE 売上 SET 金額 = ? WHERE id = ?", _
          Array(Range("B1").Value, 3))

まとめて実行して、失敗したら全部取り消す

BeginTx h
On Error GoTo 取消

Exec h, "DELETE FROM 売上 WHERE 年度 = 2023"
AppendRange h, "売上", Sheet1.Range("A2:D5000").Value

CommitTx h
Exit Sub

取消:
RollbackTx h
MsgBox "取り消しました: " & Err.Description

テーブル設計テーブルを設計する

🔒 この章以降の操作には dackdb_admin.dll（管理者）が必要です。

新しいデータベースを作る

Dim h As LongLong
h = CreateDb("C:\data\dack.db")   ' 既にファイルがあるとエラー

既存ファイルを黙って上書きしないよう、CreateDb はファイルが既にある場合エラーにします。既存のものを開くときは OpenDb です。

キーと制約をつけてテーブルを作る

制約はテーブルを作るときにまとめて書きます。あとから追加することはできないので（後述）、最初にきちんと定義しておくのが重要です。

ExecDDL h, _
    "CREATE TABLE 部署 (" & _
    "  部署コード INTEGER PRIMARY KEY, " & _
    "  部署名     VARCHAR NOT NULL" & _
    ")"

ExecDDL h, _
    "CREATE TABLE 社員 (" & _
    "  社員番号   INTEGER PRIMARY KEY, " & _
    "  氏名       VARCHAR NOT NULL, " & _
    "  メール     VARCHAR UNIQUE, " & _
    "  年齢       INTEGER CHECK (年齢 >= 0), " & _
    "  区分       VARCHAR DEFAULT '正社員', " & _
    "  部署コード INTEGER REFERENCES 部署(部署コード)" & _
    ")"
書き方意味違反したとき
PRIMARY KEY主キー。重複と NULL を許さないエラーで拒否
NOT NULL空欄を許さないエラーで拒否
UNIQUE重複を許さない（NULL は可）エラーで拒否
CHECK (条件)条件を満たす値だけ許すエラーで拒否
DEFAULT 値省略時に入る値—
REFERENCES 表(列)外部キー。参照先に無い値を許さないエラーで拒否

複数の列をまとめて主キーにするときは、列定義のあとに書きます。

ExecDDL h, _
    "CREATE TABLE 勤怠 (" & _
    "  社員番号 INTEGER, " & _
    "  日付     DATE, " & _
    "  時間     DOUBLE, " & _
    "  PRIMARY KEY (社員番号, 日付)" & _
    ")"
⚠️

「今日の日付」を既定値にするとき、CURRENT_DATE は使えません。この関数は追加拡張（ICU）を必要とし、本 DLL は外部アクセスを塞いでいるため読み込めないためです。

代わりに CAST(CAST(now() AS TIMESTAMP) AS DATE) を使ってください。

' ✗ 使えない: CURRENT_DATE / today() / now()::DATE
' ✓ 使える
ExecDDL h, "CREATE TABLE 記録 (" & _
    "  id INTEGER PRIMARY KEY, " & _
    "  登録日時 TIMESTAMP DEFAULT now(), " & _
    "  登録日   DATE DEFAULT CAST(CAST(now() AS TIMESTAMP) AS DATE)" & _
    ")"

使える日時の関数：now()、CURRENT_TIMESTAMP、get_current_timestamp()、strftime(now(), '%Y-%m-%d')、date_trunc()、DATE '2024-01-15' のようなリテラル。

連番を自動で振る

ExecDDL h, "CREATE SEQUENCE 伝票番号 START 1"
ExecDDL h, "CREATE TABLE 伝票 (" & _
    "  id   INTEGER DEFAULT nextval('伝票番号') PRIMARY KEY, " & _
    "  内容 VARCHAR" & _
    ")"

' id を省略すると自動で連番が入る
Exec h, "INSERT INTO 伝票 (内容) VALUES ('一件目'), ('二件目')"

よく使う SELECT に名前をつける（ビュー）

ExecDDL h, "CREATE OR REPLACE VIEW 部署別売上 AS " & _
    "SELECT 部署, SUM(金額) AS 合計 FROM 売上 GROUP BY 部署"

' 以降はテーブルと同じように使える
QueryToSheet h, "SELECT * FROM 部署別売上 ORDER BY 合計 DESC", Sheet1.Range("A1")

テーブル設計インデックス

🔒 インデックスの作成には dackdb_admin.dll（管理者）が必要です。 
' 1 列
ExecDDL h, "CREATE INDEX idx_社員_氏名 ON 社員 (氏名)"

' 複数列（絞り込みの順番に合わせて並べる）
ExecDDL h, "CREATE INDEX idx_勤怠 ON 勤怠 (社員番号, 日付)"

' 重複を許さないインデックス（UNIQUE 制約と同じ効果）
ExecDDL h, "CREATE UNIQUE INDEX idx_社員_メール ON 社員 (メール)"

' 式に対しても張れる
ExecDDL h, "CREATE INDEX idx_氏名小文字 ON 社員 (lower(氏名))"

' 既にあれば何もしない
ExecDDL h, "CREATE INDEX IF NOT EXISTS idx_社員_氏名 ON 社員 (氏名)"

' 削除
ExecDDL h, "DROP INDEX idx_氏名小文字"
📝

DuckDB では、インデックスは他のデータベースほど万能ではありません。列単位でデータを持つ設計のため、集計や広い範囲の検索はインデックスが無くても十分速く、実際インデックスを張っても全件走査が選ばれることがあります。

効果がはっきり出るのは UNIQUE / 主キーの重複チェックと、1 件だけを狙って取り出す検索です。「遅いから念のため全列に張る」のではなく、遅い箇所を確かめてから張ってください。

テーブル設計構造を変える

🔒 この操作には dackdb_admin.dll（管理者）が必要です。 
ExecDDL h, "ALTER TABLE 社員 ADD COLUMN 備考 VARCHAR"
ExecDDL h, "ALTER TABLE 社員 ADD COLUMN 在籍 BOOLEAN DEFAULT true"
ExecDDL h, "ALTER TABLE 社員 DROP COLUMN 備考"
ExecDDL h, "ALTER TABLE 社員 RENAME COLUMN 区分 TO 雇用区分"
ExecDDL h, "ALTER TABLE 社員 ALTER COLUMN 年齢 TYPE BIGINT"
ExecDDL h, "ALTER TABLE 社員 ALTER COLUMN 雇用区分 SET DEFAULT '正社員'"
ExecDDL h, "ALTER TABLE 社員 ALTER COLUMN 氏名 SET NOT NULL"
ExecDDL h, "ALTER TABLE 社員 RENAME TO 従業員"

' テーブル・ビューの削除
ExecDDL h, "DROP TABLE IF EXISTS 一時表"
ExecDDL h, "DROP VIEW IF EXISTS 部署別売上"
⚠️

インデックスが張られているテーブルは、列の削除・改名・テーブル名の変更ができません。

Dependency Error: Cannot alter entry ... because there are entries that depend on it. というエラーになります。

インデックスをいったん削除してから変更し、貼り直してください。

' インデックスがあるテーブルの列を消す手順
ExecDDL h, "DROP INDEX idx_社員_氏名"
ExecDDL h, "ALTER TABLE 社員 DROP COLUMN 不要列"
ExecDDL h, "CREATE INDEX idx_社員_氏名 ON 社員 (氏名)"

なお主キーだけなら変更を妨げません。明示的に作ったインデックスだけが対象です。

⚠️

制約をあとから追加することはできません。ALTER TABLE ... ADD CONSTRAINT は DuckDB が未対応で、No support for that ALTER TABLE option yet! になります。

どうしても必要なときは、新しい定義でテーブルを作り直してデータを移します。

' 制約を足したいときの作り直し手順
BeginTx h
ExecDDL h, "CREATE TABLE 社員_新 (" & _
    "  社員番号 INTEGER PRIMARY KEY, " & _
    "  氏名     VARCHAR NOT NULL, " & _
    "  年齢     INTEGER CHECK (年齢 BETWEEN 0 AND 150)" & _
    ")"
Exec    h, "INSERT INTO 社員_新 SELECT 社員番号, 氏名, 年齢 FROM 社員"
ExecDDL h, "DROP TABLE 社員"
ExecDDL h, "ALTER TABLE 社員_新 RENAME TO 社員"
CommitTx h

テーブル設計構造を調べる

' テーブルの一覧（全階層で使えます）
Dim t As Variant
t = ListTables(h)
Sheet2.Range("A1").Resize(UBound(t, 1), UBound(t, 2)).Value = t

' 1 つのテーブルの列定義（全階層で使えます）
Dim d As Variant
d = Describe(h, "社員")

ExportSchema は管理者専用で、4 つの形式を出せます。

形式返るもの内容
"table"（既定）2 次元配列全テーブルの列定義。主キーには「PK」印
"index"2 次元配列インデックス名・対象テーブル・一意かどうか・定義
"constraint"2 次元配列主キー・一意・NOT NULL・CHECK・外部キーの一覧
"ddl"文字列CREATE 文（テーブル＋インデックス）
 
Dim s As Variant
s = ExportSchema(h, "index")
Sheet3.Range("A1").Resize(UBound(s, 1), UBound(s, 2)).Value = s

' 定義をそのまま文字列で（バックアップや差分確認に）
MsgBox ExportSchema(h, "ddl")

SQL で直接調べることもできます。こちらは読み取り専用の DLL でも使えます。

QueryToSheet h, "SELECT * FROM duckdb_indexes()",     Sheet1.Range("A1")
QueryToSheet h, "SELECT * FROM duckdb_constraints()", Sheet1.Range("A1")
QueryToSheet h, "SELECT * FROM duckdb_views()",       Sheet1.Range("A1")
QueryToSheet h, "SELECT table_name, estimated_size, column_count FROM duckdb_tables()", Sheet1.Range("A1")

' 列ごとの件数・最小・最大・NULL 率などをまとめて確認
QueryToSheet h, "SUMMARIZE 社員", Sheet1.Range("A1")

' このクエリがどう処理されるか
QueryToSheet h, "EXPLAIN SELECT * FROM 社員 WHERE 氏名 = '山田'", Sheet1.Range("A1")

サンプルまるごとサンプル

ここまでの内容をひとつにまとめた、そのまま動くコードです。VBE で「挿入」→「標準モジュール」を選び、下のコードを丸ごと貼り付けて F5 を押してください。

📋

必要なもの

  • DackDbAdmin.bas をインポート済みであること（テーブル作成を含むため）
  • dackdb_admin.dll がブックと同じ場所の dll フォルダにあること
  • ブックが保存済みであること（ThisWorkbook.Path を使うため）

読み書き用・読み取り用の DLL しか無い場合は、テーブル作成の部分を飛ばして「データを入れる」以降だけを参考にしてください。

Option Explicit

'==============================================================
' dackdb 総合サンプル
'   F5 で サンプル_実行 を動かしてください。
'   Sheet1 に社員データを書き出し、Sheet2 にスキーマを出力します。
'=============================================================='

Public Sub サンプル_実行()
    Dim h As LongLong
    Dim dbPath As String

    On Error GoTo 後始末

    '--- 1. DLL を読み込む（ブックを開いたとき一度でも可）---
    DackDbInit ThisWorkbook.Path & "\dll"

    '--- 2. まっさらな DB を作る ---
    dbPath = ThisWorkbook.Path & "\sample.db"
    If Dir$(dbPath) <> "" Then Kill dbPath      ' 作り直すので消す
    h = CreateDb(dbPath)

    '--- 3. テーブルを作る（キー・制約つき）---
    ExecDDL h, _
        "CREATE TABLE 部署 (" & _
        "  部署コード INTEGER PRIMARY KEY, " & _
        "  部署名     VARCHAR NOT NULL" & _
        ")"

    ExecDDL h, _
        "CREATE TABLE 社員 (" & _
        "  社員番号   INTEGER PRIMARY KEY, " & _
        "  氏名       VARCHAR NOT NULL, " & _
        "  メール     VARCHAR UNIQUE, " & _
        "  年齢       INTEGER CHECK (年齢 BETWEEN 0 AND 150), " & _
        "  給与       DECIMAL(12,2), " & _
        "  入社日     DATE DEFAULT CAST(CAST(now() AS TIMESTAMP) AS DATE), " & _
        "  部署コード INTEGER REFERENCES 部署(部署コード)" & _
        ")"

    '--- 4. インデックスを張る ---
    ExecDDL h, "CREATE INDEX idx_社員_氏名 ON 社員 (氏名)"
    ExecDDL h, "CREATE INDEX idx_社員_部署 ON 社員 (部署コード)"

    '--- 5. マスタを入れる ---
    Exec h, "INSERT INTO 部署 VALUES (1,'営業部'), (2,'技術部'), (3,'総務部')"

    '--- 6. シートの表をまとめて投入する ---
    サンプル_見出しと明細を用意 Sheet1
    Dim 件数 As LongLong
    件数 = AppendRange(h, "社員", Sheet1.Range("A2:G4").Value)
    Debug.Print 件数 & " 件を投入しました"

    '--- 7. 1 件ずつ足す（パラメータ付きで安全に）---
    ExecP h, _
        "INSERT INTO 社員 (社員番号, 氏名, メール, 年齢, 給与, 部署コード) " & _
        "VALUES (?, ?, ?, ?, ?, ?)", _
        Array(104, "高橋 花子", "takahashi@example.jp", 29, 4200000, 2)

    '--- 8. 集計して結果をシートに貼る ---
    Sheet1.Range("J:M").ClearContents
    QueryToSheet h, _
        "SELECT d.部署名 AS 部署, " & _
        "       COUNT(*) AS 人数, " & _
        "       ROUND(AVG(s.年齢), 1) AS 平均年齢, " & _
        "       SUM(s.給与) AS 給与合計 " & _
        "FROM 社員 s JOIN 部署 d ON s.部署コード = d.部署コード " & _
        "GROUP BY d.部署名 ORDER BY 給与合計 DESC", _
        Sheet1.Range("J1")

    '--- 9. 条件をセルから受け取って検索する ---
    Dim arr As Variant
    arr = QueryP(h, _
        "SELECT 氏名, 年齢, 給与 FROM 社員 WHERE 部署コード = ? AND 年齢 >= ? ORDER BY 年齢", _
        Array(2, 25))
    Debug.Print "該当 " & (UBound(arr, 1) - 1) & " 件"

    '--- 10. 制約が効いていることを確かめる ---
    サンプル_制約を試す h

    '--- 11. スキーマを Sheet2 に出す ---
    Dim s As Variant
    Sheet2.Cells.ClearContents
    s = ExportSchema(h, "table")
    Sheet2.Range("A1").Resize(UBound(s, 1), UBound(s, 2)).Value = s
    s = ExportSchema(h, "index")
    Sheet2.Range("A1").Offset(UBound(s, 1) + 12).Resize(UBound(s, 1), UBound(s, 2)).Value = s

    MsgBox "完了しました。" & vbCrLf & _
           "Sheet1 に集計、Sheet2 にスキーマを出力しました。", vbInformation

後始末:
    If h <> 0 Then CloseDb h          ' 失敗しても必ず閉じる
    If Err.Number <> 0 Then
        MsgBox "エラーが発生しました。" & vbCrLf & vbCrLf & Err.Description, vbExclamation
    End If
End Sub


'--- 投入元の表をシートに用意する（実運用では既存の表を使ってください）---
Private Sub サンプル_見出しと明細を用意(ws As Worksheet)
    ws.Range("A1:G4").ClearContents
    ws.Range("A1:G1").Value = _
        Array("社員番号", "氏名", "メール", "年齢", "給与", "入社日", "部署コード")
    ws.Range("A2:G2").Value = _
        Array(101, "山田 太郎", "yamada@example.jp", 34, 5200000, DateSerial(2018, 4, 1), 1)
    ws.Range("A3:G3").Value = _
        Array(102, "鈴木 次郎", "suzuki@example.jp", 41, 6800000, DateSerial(2012, 10, 1), 2)
    ws.Range("A4:G4").Value = _
        Array(103, "佐藤 三郎", "sato@example.jp", 27, 3900000, DateSerial(2022, 4, 1), 3)
End Sub


'--- 制約が本当に効くか、わざと違反して確かめる ---
Private Sub サンプル_制約を試す(ByVal h As LongLong)
    Debug.Print 試行(h, "主キー重複", _
        "INSERT INTO 部署 VALUES (1, '重複部署')")
    Debug.Print 試行(h, "NOT NULL 違反", _
        "INSERT INTO 部署 VALUES (9, NULL)")
    Debug.Print 試行(h, "CHECK 違反", _
        "INSERT INTO 社員 (社員番号, 氏名, 年齢) VALUES (900, 'テスト', 999)")
    Debug.Print 試行(h, "外部キー違反", _
        "INSERT INTO 社員 (社員番号, 氏名, 部署コード) VALUES (901, 'テスト', 99)")
End Sub

'--- 実行してみて、拒否されたかどうかを文字列で返す ---
Private Function 試行(ByVal h As LongLong, ByVal 名前 As String, _
                        ByVal sql As String) As String
    On Error Resume Next
    Exec h, sql
    If Err.Number <> 0 Then
        試行 = 名前 & ": 期待どおり拒否されました"
        Err.Clear
    Else
        試行 = 名前 & ": ★通ってしまいました（制約を確認してください）"
    End If
    On Error GoTo 0
End Function

実行結果のうち Debug.Print の出力は、VBE の イミディエイト ウィンドウ（Ctrl + G）に表示されます。制約が 4 つとも「期待どおり拒否されました」と出れば、キーと制約が正しく効いています。

Public Sub 日次処理()

    On Error GoTo 後始末

    h = OpenDb("C:\data\dack.db")

    AppendRange h, "売上", Sheet1.Range("A2:D5000").Value
    CommitTx    h

後始末:
    If h <> 0 Then CloseDb h

困ったときエラーが出たら

エラーメッセージは日本語で、対処方法まで書かれています。よく出るものを挙げます。

DLL が見つかりません: ...\dackdb_r.dll

DackDbInit に渡したフォルダに DLL が入っていません。フォルダのパスと、DLL のファイル名が一致しているか確認してください。

ブックと同じ場所の dll フォルダに置いた場合は DackDbInit ThisWorkbook.Path & "\dll" です。

このモジュールは 64bit 版 Excel 専用です

32bit 版の Excel では動きません。「ファイル」→「アカウント」→「Excel のバージョン情報」でご確認のうえ、情報システム部門にご相談ください。

【権限エラー】DackExecute はこの DLL では使えません

読み取り専用の DLL でデータを書き込もうとしています。書き込みには dackdb_rw.dll、テーブルの作成には dackdb_admin.dll が必要です。

メッセージの続きに、必要な DLL の名前が書かれています。配布元にご相談ください。

データベースファイルが見つかりません: C:\data\dack.db

パスが間違っているか、ファイルがまだ作られていません。読み取り専用の DLL は新しいファイルを作れないので、その場合は管理者の DLL で CreateDb を使う必要があります。

IO Error: ... 別のプロセスが使用中です

同じ Excel の中で、同じファイルを 2 回開こうとしています。これはできません。

先に開いたほうを CloseDb してから、開き直してください。前のマクロがエラーで途中終了して閉じ忘れている、というのがよくある原因です。Excel を再起動すれば必ず解消します。

接続ハンドル 0 は無効です

OpenDb が成功していないか、既に CloseDb したハンドルを使っています。OpenDb の行より前で処理が止まっていないか確認してください。

この DLL は読み書き可（READ_WRITE）です。DROP（削除）は実行できません

テーブルそのものを消す・作る・変える操作には管理者の DLL が必要です。データの削除（DELETE）は読み書き可の DLL でできます。

3 行 2 列目: セルがエラー値（#N/A、#DIV/0! など）です

AppendRange に渡した範囲にエラー値のセルがあります。示された位置のセルを直すか、IFERROR で空欄にしてから渡してください。

エラー値を自動で空欄に変換していないのは、気づかないうちにデータが欠けるのを防ぐためです。

テーブル「売上」は 4 列ですが、渡された範囲は 3 列です

AppendRange に渡す範囲の列数を、テーブルの列数に合わせてください。Describe(h, "売上") で列の並び順を確認できます。見出し行を含めてしまっていないかも確認してください。

結果が 2000000 行あり、Excel シートの上限を超えています

SQL に LIMIT を付けて絞り込むか、WHERE で条件を足してください。集計してから取り出す（GROUP BY）のも有効です。

Dependency Error: Cannot alter entry ... because there are entries that depend on it

インデックスが張られているテーブルの列を削除・改名しようとしています。DuckDB はこれを許しません。

インデックスをいったん削除してから変更し、貼り直してください。

ExecDDL h, "DROP INDEX idx_社員_氏名"
ExecDDL h, "ALTER TABLE 社員 DROP COLUMN 不要列"
ExecDDL h, "CREATE INDEX idx_社員_氏名 ON 社員 (氏名)"

どのインデックスがあるかは ExportSchema(h, "index") で確認できます。

Not implemented Error: No support for that ALTER TABLE option yet!

ALTER TABLE ... ADD CONSTRAINT のように、制約をあとから追加しようとしています。DuckDB は未対応です。

制約はテーブルを作るときに書く必要があります。既存のテーブルに足したい場合は、新しい定義でテーブルを作ってデータを移し替えてください（「構造を変える」の章に手順があります）。

Scalar Function with name "current_date" is not in the catalog, but it exists in the icu extension

CURRENT_DATE や today() は追加拡張（ICU）を必要とします。この DLL は安全のため外部アクセスを塞いでいるので、拡張を読み込めません。

次のように書き換えてください。

使えない代わりに
CURRENT_DATECAST(CAST(now() AS TIMESTAMP) AS DATE)
today()同上
now()::DATE同上（now() はタイムゾーン付きのため直接は変換できません）

now()、CURRENT_TIMESTAMP、strftime()、date_trunc() はそのまま使えます。

Conversion Error: Unimplemented type for cast (TIMESTAMP WITH TIME ZONE -> DATE)

now()::DATE や CAST(now() AS DATE) を実行しています。now() はタイムゾーン付きの日時を返すため、日付へ直接変換するには ICU 拡張が必要です。

いったんタイムゾーン無しに落としてから変換してください。

CAST(CAST(now() AS TIMESTAMP) AS DATE)
Constraint Error: Duplicate key ... violates primary key constraint

主キーが重複しています。制約が正しく働いている証拠なので、データ側を直してください。

「既にあれば更新、無ければ追加」をしたい場合は UPSERT が使えます。

Exec h, "INSERT INTO 在庫 VALUES ('A', 25) " & _
        "ON CONFLICT (品番) DO UPDATE SET 数量 = EXCLUDED.数量"
SQL の構文エラー: ...

SQL の書き方に誤りがあります。メッセージの後半にデータベースからの指摘がそのまま入っているので、そこを手がかりに直してください。日本語のテーブル名や列名は、そのまま書けます。

最後に知っておくこと

同じファイルは同時に開けません

1 つの Excel の中で、同じ dack.db を 2 つのハンドルで開くことはできません。必ず片方を閉じてから開いてください。別の Excel（別プロセス）から読み取り目的で開くことは可能です。

DLL の分割は「うっかり」を防ぐためのものです

読み取り専用の DLL を配ることで、操作ミスによるデータの破損を防げます。ただしこれはセキュリティのための仕組みではありません。管理者用の DLL を入手した人は何でもできます。厳密に制限する必要がある場合は、データベースファイル自体に Windows の読み取り専用権限を設定してください。

大きなデータの扱い方

DuckDB 側は数千万行を扱えますが、Excel のシートは 104 万行までです。全件を取り出すのではなく、SQL で絞り込む・集計してから取り出すのがこのモジュールの本来の使い方です。

' 全件取り出してから Excel で集計する（遅い・上限に当たる）
arr = Query(h, "SELECT * FROM 売上")

' データベース側で集計してから取り出す（速い）
arr = Query(h, "SELECT 部署, SUM(金額) AS 合計 " & _
                "FROM 売上 GROUP BY 部署 ORDER BY 合計 DESC")
