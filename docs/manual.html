dackdb 使い方マニュアル

Excel の VBA から DuckDB のデータベース（dack.db）を読み書きするためのモジュールです。数十万行を 1 秒かからずシートに展開できます。

対象 64bit 版 Excel 用意するもの DLL 1 ファイルと .bas 1 ファイル 前提知識 VBA の基本

はじめにこれは何か

DuckDB は 1 つのファイルにデータを収める分析用データベースです。Excel の 104 万行の壁を超える量のデータを置いておき、必要な分だけ SQL で絞り込んでシートに取り出す、という使い方ができます。

このモジュールは、その DuckDB を VBA から直接呼び出せるようにするものです。配布された DLL によって、あなたができる操作の範囲が決まります。

💡

日本語がそのまま扱えます。テーブル名・列名・データのいずれにも日本語を使えて、文字化けしません。𠮷 のようなサロゲートペアや絵文字も含みます。ODBC 接続を使った方法だとここが壊れますが、このモジュールでは起きません。

ステップ 1準備する

初回だけ必要な作業です。3 つのステップで終わります。

  1. ファイルを置く

    配布された DLL を、ブックと同じ場所の dll フォルダに入れます。フォルダ名や場所は自由ですが、以下の説明はこの配置を前提にします。

    売上管理.xlsm
    dll\dackdb_r.dll　　← 配布された DLL を 1 つ

  2. モジュールを読み込む

    Excel で Alt + F11 を押して VBE を開き、メニューの「ファイル」→「ファイルのインポート」から、DLL に対応する .bas ファイルを選びます。

    配布された DLL読み込む .bas
    dackdb_r.dllDackDbR.bas
    dackdb_rw.dllDackDbRW.bas
    dackdb_admin.dllDackDbAdmin.bas

  3. ブックを開いたときに初期化する

    VBE の左側で「ThisWorkbook」をダブルクリックし、次のコードを貼り付けます。ブックを開くたびに自動で実行されます。

    Private Sub Workbook_Open()
    DackDbInit ThisWorkbook.Path & "\dll"
End Sub

    これで準備完了です。ブックを保存して開き直してください。

⚠️

32bit 版の Excel では動きません。「ファイル」→「アカウント」→「Excel のバージョン情報」で確認できます。32bit 版だった場合は初期化時にその旨のエラーが出ます。

ステップ 2自分の権限を確認する

DLL は 3 種類あり、できる操作が違います。下のボタンで、あなたが配布された DLL を選んでください。このページ全体が、その権限でできることに合わせて表示されます。

分からない場合は、次のコードを実行すると自分の権限が表示されます。

Sub 権限を確認()
    MsgBox DackVersion()
End Sub

' 表示例:
' dackdb 0.1.0 / DuckDB v1.5.5 / 権限: READ_WRITE (読み書き可（DDL 不可）)
📝

権限外の操作を書いても、VBA のコードとしては正しく書けます。実行したときに「どの DLL を使えばよいか」を伝えるエラーが出ます。上位の DLL を渡されれば、コードを 1 文字も直さずそのまま動きます。

ステップ 3データを取り出す

いちばん短い例

データベースを開き、SQL の結果をシートに貼り、閉じる。これが基本形です。

Sub 営業部の売上を取得()
    Dim h As LongLong
    h = OpenDb("C:\data\dack.db")

    Dim n As Long
    n = QueryToSheet(h, "SELECT * FROM 売上 WHERE 部署 = '営業部'", _
                     Sheet1.Range("A1"))

    CloseDb h
    MsgBox n & " 件貼り付けました"
End Sub

QueryToSheet は 1 行目に列名、2 行目以降にデータを貼り付け、データの行数（列名の行を除いた数）を返します。10 万行でも一瞬で終わります。

🔒

CloseDb を忘れないでください。閉じないとファイルがロックされたままになり、他の人がそのデータベースを開けなくなります。エラーが起きても必ず閉じたい場合は、下の「安全な書き方」を参考にしてください。

配列として受け取る

シートに貼らず、VBA の中で処理したいときは Query を使います。

Dim arr As Variant
arr = Query(h, "SELECT 部署, 金額 FROM 売上")

' arr(1, 1) = "部署"　　← 1 行目は必ず列名
' arr(1, 2) = "金額"
' arr(2, 1) = "営業部"　← 2 行目からがデータ
' arr(2, 2) = 1000

Dim i As Long, 合計 As Double
For i = 2 To UBound(arr, 1)
    If Not IsNull(arr(i, 2)) Then 合計 = 合計 + arr(i, 2)
Next

データが 0 件のときは列名の行だけが返るので、UBound(arr, 1) = 1 で判定できます。データベースの NULL は IsNull() で判定でき、シートに貼ると空欄になります。

シートの値を条件に使うとき

🛡️

SQL に文字列を連結しないでください。セルに ' OR 1=1; DROP TABLE 売上; -- のような値が入っていると、意図しない SQL が実行されてしまいます。

代わりに QueryP を使い、値を ? の位置に渡します。渡した値は必ず「ただの値」として扱われ、SQL として解釈されることはありません。

' ✗ 危険：セルの内容が SQL の一部になってしまう
arr = Query(h, "SELECT * FROM 売上 WHERE 部署 = '" & Range("B1").Value & "'")

' ✓ 安全：? に値として渡る
arr = QueryP(h, "SELECT * FROM 売上 WHERE 部署 = ? AND 金額 > ?", _
             Array(Range("B1").Value, 1000))

渡し方は Array(値1, 値2) でも Range("A1:C1").Value でも構いません。縦並びの範囲（Range("A1:A3").Value）も受け付けます。

安全な書き方（エラーが起きても必ず閉じる）

Sub 売上を取得()
    Dim h As LongLong
    On Error GoTo 後始末

    h = OpenDb("C:\data\dack.db")
    QueryToSheet h, "SELECT * FROM 売上", Sheet1.Range("A1")

後始末:
    If h <> 0 Then CloseDb h
    If Err.Number <> 0 Then MsgBox Err.Description, vbExclamation
End Sub

ステップ 4データを書き込む

🔒 読み取り専用の DLL では、この章の操作はできません。dackdb_rw.dll が必要です。

シートの範囲をまとめて追加する

Excel から DB へデータを入れる、いちばん速い方法です。2 万行で 0.04 秒ほどで終わります。

Dim n As LongLong
n = AppendRange(h, "売上", Sheet1.Range("A2:D10000").Value)

使うときの決まりは 3 つです。

  • 見出しの行は含めない。データの行だけの範囲を渡します（上の例では 1 行目が見出しなので A2 から）。
  • 列の並び順をテーブルに合わせる。1 列目がテーブルの 1 列目に入ります。列数が違うとエラーになります。
  • 途中で失敗したら 1 行も入りません。「1000 行目でエラー、でも 999 行は入った」という中途半端な状態にはなりません。
⚠️

範囲に #N/A や #DIV/0! などのエラー値のセルがあると、その位置を示してエラーになります（例:「3 行 2 列目: セルがエラー値です」）。空欄として扱いたい場合は IFERROR で空にしてから渡してください。エラー値を勝手に空欄に変えてしまうと、気づかないうちにデータが壊れるためです。

1 件ずつ追加・更新・削除する

' 影響した行数が返る
Dim n As LongLong

n = Exec(h, "DELETE FROM 売上 WHERE 日付 < DATE '2020-01-01'")

' セルの値を使うときは ExecP（? に値が入る）
n = ExecP(h, "UPDATE 売上 SET 金額 = ? WHERE id = ?", _
          Array(Range("B1").Value, 3))

まとめて実行して、失敗したら全部取り消す

BeginTx h
On Error GoTo 取消

Exec h, "DELETE FROM 売上 WHERE 年度 = 2023"
AppendRange h, "売上", Sheet1.Range("A2:D5000").Value

CommitTx h
Exit Sub

取消:
RollbackTx h
MsgBox "取り消しました: " & Err.Description

ステップ 5テーブルを作る・調べる

🔒 この章の操作には dackdb_admin.dll（管理者）が必要です。

新しいデータベースを作る

Dim h As LongLong
h = CreateDb("C:\data\新規.db")   ' 既にファイルがあるとエラー

ExecDDL h, "CREATE TABLE 売上 (" & _
    "id INTEGER PRIMARY KEY, " & _
    "部署 VARCHAR NOT NULL, " & _
    "金額 BIGINT, " & _
    "日付 DATE)"

CloseDb h

既存ファイルを黙って上書きしないよう、CreateDb はファイルが既にある場合エラーにします。開き直したいときは OpenDb を使ってください。

今の構造を調べる

' テーブルの一覧
Sheet2.Range("A1").Resize(100, 3).Value = ListTables(h)

' 1 つのテーブルの列定義
Dim d As Variant
d = Describe(h, "売上")

' 全テーブルの列＋主キー情報をシートに（管理者のみ）
Dim s As Variant
s = ExportSchema(h, "table")
Sheet3.Range("A1").Resize(UBound(s, 1), UBound(s, 2)).Value = s

' CREATE 文をそのまま文字列で取り出す
MsgBox ExportSchema(h, "ddl")

参考型の対応

データベースの型が Excel でどう見えるかの一覧です。ふだんは意識しなくて構いません。

BOOLEANBoolean
INTEGER などLong
BIGINTLongLong

LIST や STRUCT のような入れ子の型は、セルに入れられないためエラーになります。SQL で文字列に変換すれば取り出せます。

困ったときエラーが出たら

エラーメッセージは日本語で、対処方法まで書かれています。よく出るものを挙げます。

DLL が見つかりません: ...\dackdb_r.dll

DackDbInit に渡したフォルダに DLL が入っていません。フォルダのパスと、DLL のファイル名が一致しているか確認してください。

ブックと同じ場所の dll フォルダに置いた場合は DackDbInit ThisWorkbook.Path & "\dll" です。

このモジュールは 64bit 版 Excel 専用です

32bit 版の Excel では動きません。「ファイル」→「アカウント」→「Excel のバージョン情報」でご確認のうえ、情報システム部門にご相談ください。

【権限エラー】DackExecute はこの DLL では使えません

読み取り専用の DLL でデータを書き込もうとしています。書き込みには dackdb_rw.dll、テーブルの作成には dackdb_admin.dll が必要です。

メッセージの続きに、必要な DLL の名前が書かれています。配布元にご相談ください。

データベースファイルが見つかりません: C:\data\dack.db

パスが間違っているか、ファイルがまだ作られていません。読み取り専用の DLL は新しいファイルを作れないので、その場合は管理者の DLL で CreateDb を使う必要があります。

IO Error: ... 別のプロセスが使用中です

同じ Excel の中で、同じファイルを 2 回開こうとしています。これはできません。

先に開いたほうを CloseDb してから、開き直してください。前のマクロがエラーで途中終了して閉じ忘れている、というのがよくある原因です。Excel を再起動すれば必ず解消します。

接続ハンドル 0 は無効です

OpenDb が成功していないか、既に CloseDb したハンドルを使っています。OpenDb の行より前で処理が止まっていないか確認してください。

この DLL は読み書き可（READ_WRITE）です。DROP（削除）は実行できません

テーブルそのものを消す・作る・変える操作には管理者の DLL が必要です。データの削除（DELETE）は読み書き可の DLL でできます。

3 行 2 列目: セルがエラー値（#N/A、#DIV/0! など）です

AppendRange に渡した範囲にエラー値のセルがあります。示された位置のセルを直すか、IFERROR で空欄にしてから渡してください。

エラー値を自動で空欄に変換していないのは、気づかないうちにデータが欠けるのを防ぐためです。

テーブル「売上」は 4 列ですが、渡された範囲は 3 列です

AppendRange に渡す範囲の列数を、テーブルの列数に合わせてください。Describe(h, "売上") で列の並び順を確認できます。見出し行を含めてしまっていないかも確認してください。

結果が 2000000 行あり、Excel シートの上限を超えています

SQL に LIMIT を付けて絞り込むか、WHERE で条件を足してください。集計してから取り出す（GROUP BY）のも有効です。

SQL の構文エラー: ...

SQL の書き方に誤りがあります。メッセージの後半にデータベースからの指摘がそのまま入っているので、そこを手がかりに直してください。日本語のテーブル名や列名は、そのまま書けます。

最後に知っておくこと

同じファイルは同時に開けません

1 つの Excel の中で、同じ dack.db を 2 つのハンドルで開くことはできません。必ず片方を閉じてから開いてください。別の Excel（別プロセス）から読み取り目的で開くことは可能です。

DLL の分割は「うっかり」を防ぐためのものです

読み取り専用の DLL を配ることで、操作ミスによるデータの破損を防げます。ただしこれはセキュリティのための仕組みではありません。管理者用の DLL を入手した人は何でもできます。厳密に制限する必要がある場合は、データベースファイル自体に Windows の読み取り専用権限を設定してください。

大きなデータの扱い方

DuckDB 側は数千万行を扱えますが、Excel のシートは 104 万行までです。全件を取り出すのではなく、SQL で絞り込む・集計してから取り出すのがこのモジュールの本来の使い方です。

' 全件取り出してから Excel で集計する（遅い・上限に当たる）
arr = Query(h, "SELECT * FROM 売上")

' データベース側で集計してから取り出す（速い）
arr = Query(h, "SELECT 部署, SUM(金額) AS 合計 " & _
                "FROM 売上 GROUP BY 部署 ORDER BY 合計 DESC")
